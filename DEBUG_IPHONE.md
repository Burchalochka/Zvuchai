# Инструкция по запуску на реальном iPhone

## Что было исправлено:

1. ✅ Исправлен `RCTNewArchEnabled` в Info.plist (теперь `false`, соответствует Podfile)
2. ✅ Убрано ограничение `UIRequiredDeviceCapabilities` с `armv7`
3. ✅ Добавлена автоматическая подпись кода (`CODE_SIGN_STYLE = Automatic`)

## Шаги для запуска на iPhone:

### 1. Откройте проект в Xcode:
```bash
cd ios
open Zvuchai_TEMP.xcworkspace
```

### 2. Настройте подпись:
- В Xcode выберите проект "Zvuchai_TEMP" (синяя иконка слева)
- Выберите target "Zvuchai_TEMP"
- Перейдите на вкладку **"Signing & Capabilities"**
- Включите **"Automatically manage signing"**
- Выберите ваш **Team** (ваш Apple ID)

### 3. Подключите iPhone:
- Подключите iPhone к Mac через USB
- Разблокируйте iPhone
- На iPhone появится запрос "Доверять этому компьютеру?" → нажмите **"Доверять"**

### 4. Выберите устройство:
- В Xcode вверху рядом с кнопкой "Play" выберите ваш iPhone из списка устройств

### 5. Запустите приложение:
- Нажмите кнопку **▶️ Play** в Xcode
- Или выполните: `npx react-native run-ios --device "Ваше имя iPhone"`

## Если приложение не запускается:

### Проверьте логи устройства:
1. В Xcode: **Window → Devices and Simulators**
2. Выберите ваш iPhone
3. Нажмите **"View Device Logs"**
4. Найдите ошибки (красные сообщения)

### Возможные проблемы:

1. **Ошибка подписи:**
   - Убедитесь, что выбран правильный Team
   - Проверьте, что Bundle Identifier уникален (может потребоваться изменить `com.zvuchai.temp`)

2. **Приложение устанавливается, но не открывается:**
   - На iPhone: **Настройки → Основные → Управление устройством**
   - Найдите ваше приложение и нажмите **"Доверять"**

3. **Metro bundler не подключается:**
   - Убедитесь, что iPhone и Mac в одной Wi-Fi сети
   - Или используйте USB-подключение с настройкой в Xcode

## Альтернативный способ (через командную строку):

```bash
# Узнайте UDID вашего iPhone
xcrun xctrace list devices

# Запустите на конкретном устройстве
npx react-native run-ios --device "UDID или имя устройства"
```
